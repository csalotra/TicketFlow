TicketFlow is a **RESTful API** built with **FastAPI** and **SQLite**, designed as a **practice/refresher project** for building modern backend systems.  

The project demonstrates **JWT authentication, role-based access, and CRUD operations** in a structured FastAPI application. It is intentionally iterative, with plans for **future enhancements**, including AI-assisted ticket management.

---

## Highlights

- User registration and JWT-based authentication
- Role-based access (Users vs Admins)
- Users can create tickets; admins can view/update all tickets
- Modern REST patterns: proper HTTP codes, error handling
- Designed for **extensibility**: Docker, cloud deployment, AI features in the future

---

## Tech Stack

- **Backend:** FastAPI
- **Database:** SQLite (via SQLModel)
- **Authentication:** JWT
- **Password Security:** Passlib (bcrypt)
- **Future Plans:** AI enhancements, Dockerization, real-time notifications
